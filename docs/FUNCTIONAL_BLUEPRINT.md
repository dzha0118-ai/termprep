# TermPrep 功能蓝图

基于专业翻译译前准备流程（ISO 17100 标准 + 行业最佳实践）

## 流程总览

```
源文本输入 → [项目分析] → [术语提取] → [关联搜索] → [术语库] → [导出准备包]
                 ↓              ↓            ↓           ↓
              领域识别      频率统计      在线词典     CSV/Excel
              字数统计      关键词权重    搭配例句     术语表
              难度评估      自动分组      平行文本    记忆库
```

---

## 阶段一：项目分析 (Project Analysis)

目标：输入文本 → 自动分析翻译任务的范围和难度

| 功能模块 | 输入 | 输出 | 优先级 |
|---------|------|------|--------|
| 语言检测 | 源文本 | 源语言/目标语言对 | P0 |
| 领域识别 | 源文本 | 行业分类（法律/医学/IT/金融等） | P1 |
| 字数统计 | 源文本 | 字数/字符数/段落数 | P0 |
| 难度评估 | 源文本 | 难度等级（术语密度、句子复杂度） | P2 |

---

## 阶段二：术语提取 (Term Extraction)

目标：从源文本自动提取关键术语和重复词汇

| 功能模块 | 技术方案 | 优先级 |
|---------|---------|--------|
| 高频词统计 | jieba 分词 + TF-IDF | P0 |
| N-gram 提取 | 2-5 字词组自动识别 | P0 |
| 专有名词识别 | 正则 + 命名实体识别 | P1 |
| 术语去重归一化 | 编辑距离 + 词形还原 | P1 |
| 原文行定位 | 每个术语标注在原文中的位置 | P2 |

---

## 阶段三：关联词汇搜索 (Association Search)

目标：为每个术语找到关联词、搭配、反义词、例句

| 数据源 | 内容 | 接入方式 | 优先级 |
|--------|------|---------|--------|
| 本地词库 | SQLite 内置词库 | 直接查询 | P0 |
| 有道词典 | 中英释义+例句 | API | P1 |
| Merriam-Webster | 英文释义+同反义词 | API | P1 |
| 百度百科/Wikipedia | 概念解释 | 网页抓取 | P2 |
| 语料库 | 真实例句 | 本地索引 | P3 |

### 关联类型

```
术语 "人工智能"
  ├── 直译: artificial intelligence
  ├── 缩写: AI
  ├── 相关词: 机器学习, 深度学习, 神经网络
  ├── 搭配: 人工智能技术, 人工智能应用
  ├── 反义: 人工操作 (manual operation)
  └── 例句: "人工智能正在改变翻译行业..."
```

---

## 阶段四：术语库管理 (TermBase Management)

目标：构建可复用、可进化的个人术语库

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 创建术语库 | 按领域/项目创建 SQLite 库 | P0 |
| 导入术语 | CSV/Excel/TBX 格式导入 | P0 |
| 导出术语 | CSV/TBX/Excel 导出 | P0 |
| 术语版本控制 | Git 跟踪术语库变更（diff 可视化） | P2 |
| 术语合并 | 多个术语库去重合并 | P2 |
| 术语审批 | 标记术语状态（草稿/确认/废弃） | P2 |
| 搜索过滤 | 按领域、语言、类型筛选 | P1 |

---

## 阶段五：参考资料收集 (Reference Collection)

目标：自动收集项目相关的参考资料

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 风格指南模板 | 预置通用风格指南框架 | P2 |
| 平行文本搜索 | 搜索类似文档作为参考 | P3 |
| 客户术语表导入 | 导入甲方提供的术语表 | P1 |
| 项目笔记 | Markdown 记录翻译决策 | P2 |

---

## 阶段六：导出 & 集成 (Export & Integration)

目标：输出可直接用于翻译工具的术语包

| 格式 | 用途 | 优先级 |
|------|------|--------|
| CSV | 通用/Excel 查看 | P0 |
| TBX | 导入 CAT 工具（Trados/MemoQ） | P1 |
| XLSX | 带格式打印 | P1 |
| JSON | API/程序对接 | P0 |
| 双语对照表 | Word 文档 | P2 |

---

## CLI 命令规划

```
termprep analyze <file>           # 项目分析：语言、领域、字数
termprep extract <file>           # 术语提取：高频词、关键词
termprep search <term>            # 关联搜索：同义词、搭配、例句
termprep batch <file>             # 批量查询
termprep db init <name>           # 创建术语库
termprep db import <csv>          # 导入术语
termprep db export --format csv   # 导出术语库
termprep db merge <db1> <db2>     # 合并术语库
termprep export --format tbx      # 导出为 CAT 工具格式
termprep report                   # 生成译前准备报告
```

## 技术架构

```
termprep/
├── cli.py              # Click CLI 入口
├── analyzer.py         # 项目分析（语言检测、字数统计）
├── extractor.py        # 术语提取（jieba 分词、N-gram）
├── searcher.py         # 关联搜索（本地 + 在线 API）
├── db.py               # SQLite 术语库管理
├── exporter.py         # 多格式导出（CSV/TBX/XLSX）
├── sources/            # 在线数据源
│   ├── youdao.py       # 有道词典 API
│   ├── webster.py      # 韦氏词典 API
│   └── wikipedia.py    # 百科抓取
└── utils/
    ├── text.py         # 文本处理工具
    └── lang.py         # 语言检测工具
```
